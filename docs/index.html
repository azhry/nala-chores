Kubernetes agent runner

Delegate coding tasks from anywhere.

Nala Chores starts configured coding-agent sessions from a phone or laptop, runs them inside sandboxed Minikube jobs, and keeps the transcript, branch, and pull request ready for later review.

Built for the moment when a coding task needs to move forward from a phone, without keeping a laptop terminal open.

Set up the project once. Launch sessions when needed.

Profile

Each repository keeps its own run settings.

Save repo URL, branch, work directory, GitHub, Linear, OpenCode or KiloCode keys, sandbox size, harness URL, and config path for the project.

Session

Starting work is just profile, issue, prompt.

Pick a configuration, attach the issue key, write the task, and let the manager create the sandbox job remotely.

History

The run stays inspectable after launch.

Inspect runner milestones, agent output, tool calls, failures, job names, branch names, and pull-request URLs from the session history.

Remote delegation without starting from zero every time.

  • Profiles remember the boring setup. Credentials, harnesses, model choices, repo paths, and sandbox size are saved.
  • Runs happen inside disposable Kubernetes jobs. The agent gets a clean workspace for the task instead of touching a local checkout.
  • History keeps the session usable from mobile. Reopen the run, read the transcript, and jump to the PR later.
