The loop

A runner that proves work happened.

Nala Chores is built around one rule: a successful agent session must leave behind a pushed branch, a pull request URL, and enough logs to understand what happened.

01

Save a project profile

Store GitHub, Linear, OpenCode, KiloCode, repo, branch, harness URL, and sandbox size.

02

Run an agent session

Select a configuration, write the task prompt, attach a Linear issue key, and launch.

03

Watch the transcript

Read agent messages, tool calls, runner milestones, failures, pushes, and PR creation.

04

Verify the PR

Runs with PR creation enabled cannot succeed without committed changes and a PR URL.

Interface

Dense, inspectable, mobile-ready operations.

Session details

Chat-style logs separate agent reasoning, tool output, errors, and runner milestones so long sessions stay readable.

Runner guarantees

No more fake green checks.

The runner now rejects PR-enabled sessions that produce no repository changes. If the agent finishes without a Git diff, Nala Chores sends it back for a real filesystem fix.

Local deploy

Start it in Minikube.

minikube start --cpus 6 --memory 12288 --driver docker
kubectl apply -f deploy/minikube/namespace.yaml
kubectl apply -f deploy/minikube/rbac.yaml
kubectl apply -f deploy/minikube/runner-manager.yaml
kubectl -n agent-runner port-forward svc/runner-manager 8080:8080
