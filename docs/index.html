Built for teams that want agent sessions to feel observable, repeatable, and reviewable before the work reaches a repository.

Three surfaces. One controlled loop.

Profile

Credentials and harnesses stay with the project.

Save repo URL, branch, work directory, GitHub, Linear, OpenCode or KiloCode keys, sandbox size, harness URL, and config path once.

Session

The prompt field is intentionally narrow.

Pick a configuration, attach the issue key, write the task, and let the manager render a Kubernetes Job instead of improvising from a shell.

History

Logs read like a transcript, not raw JSON exhaust.

Inspect runner milestones, agent output, tool calls, failures, job names, branch names, and pull-request URLs from the run ledger.

Task scope stays explicit from launch to pull request.

  • Linear context is loaded before the agent starts. The session gets the issue details it needs before editing begins.
  • PR-enabled jobs report the branch and pull request. The result is easy to inspect from GitHub and the run history.
  • The requested task is repeated in the runner contract. That keeps the agent oriented toward the prompt and issue context.
